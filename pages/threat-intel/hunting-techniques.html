Stacking (Frequency Analysis)

Stack and count to identify rare occurrences that may indicate malicious activity.

KQL - Stacking Examples 
// Stack: Rare parent-child process relationships
DeviceProcessEvents
| where TimeGenerated > ago(7d)
| summarize Count = count() by 
    InitiatingProcessFileName, 
    FileName
| where Count < 5  // Rare combinations
| order by Count asc

// Stack: Unusual service installations
SecurityEvent
| where EventID == 7045  // Service installed
| summarize Count = count() by ServiceName, ServiceFileName
| where Count == 1  // Unique services
| order by Count asc

// Stack: Rare scheduled tasks
DeviceEvents
| where ActionType == "ScheduledTaskCreated"
| summarize Count = count() by TaskName = extractjson("$.TaskName", AdditionalFields)
| where Count < 3

// Stack: Uncommon PowerShell commands
DeviceProcessEvents
| where FileName == "powershell.exe"
| extend Command = extract(@"-[eE]n?c?\s+([A-Za-z0-9+/=]+)", 1, ProcessCommandLine)
| where isnotempty(Command)
| summarize Count = count() by Command
| where Count == 1  // Unique encoded commands

Baselining

Establish normal behavior and hunt for deviations.

KQL - Baseline Comparison 
// Baseline: Normal logon hours for user
let UserBaseline = SigninLogs
| where TimeGenerated > ago(30d) and TimeGenerated < ago(1d)
| extend Hour = datetime_part("Hour", TimeGenerated)
| summarize 
    NormalHours = make_set(Hour),
    AvgLogonsPerDay = count() / 30
    by UserPrincipalName;

// Hunt: Logons outside normal hours
SigninLogs
| where TimeGenerated > ago(1d)
| extend Hour = datetime_part("Hour", TimeGenerated)
| join kind=inner UserBaseline on UserPrincipalName
| where Hour !in (NormalHours)
| project TimeGenerated, UserPrincipalName, Hour, NormalHours, IPAddress

// Baseline: Normal network destinations
let NetworkBaseline = DeviceNetworkEvents
| where TimeGenerated > ago(30d)
| summarize ConnectionCount = count() by RemoteUrl
| where ConnectionCount > 100;  // Frequently accessed

// Hunt: New/rare destinations
DeviceNetworkEvents
| where TimeGenerated > ago(1d)
| summarize TodayCount = count() by RemoteUrl
| join kind=leftanti NetworkBaseline on RemoteUrl
| where TodayCount > 10  // Some volume but never seen before
| order by TodayCount desc

TTP-Based Hunting

Hunt for specific attack techniques regardless of IOCs.

KQL - Behavioral Pattern Hunting 
// Hunt: LSASS Access (Credential Dumping - T1003.001)
DeviceProcessEvents
| where TimeGenerated > ago(24h)
| where FileName =~ "lsass.exe"
| join kind=inner (
    DeviceEvents
    | where ActionType == "ProcessAccess"
    | where TargetProcessName =~ "lsass.exe"
) on DeviceId, $left.ProcessId == $right.TargetProcessId
| where InitiatingProcessFileName !in ("csrss.exe", "services.exe", "svchost.exe")
| project TimeGenerated, DeviceName, InitiatingProcessFileName, InitiatingProcessCommandLine

// Hunt: Living Off the Land (LOLBins)
let LOLBins = dynamic(["certutil.exe", "bitsadmin.exe", "mshta.exe", "regsvr32.exe", 
                        "rundll32.exe", "wmic.exe", "msbuild.exe", "installutil.exe"]);
DeviceProcessEvents
| where TimeGenerated > ago(24h)
| where FileName in~ (LOLBins)
| where ProcessCommandLine has_any ("http", "ftp", "/download", "-decode", "-urlcache")
| project TimeGenerated, DeviceName, FileName, ProcessCommandLine, AccountName

// Hunt: Lateral Movement via WMI (T1047)
DeviceProcessEvents
| where TimeGenerated > ago(24h)
| where InitiatingProcessFileName =~ "wmiprvse.exe"
| where FileName in~ ("cmd.exe", "powershell.exe", "pwsh.exe")
| project TimeGenerated, DeviceName, FileName, ProcessCommandLine

Long Tail Analysis

ðŸ’¡ The Long Tail Principle

In most environments, a small number of activities account for most events (the "head"), while malicious activity often hides in the rare occurrences (the "tail").

  • Head: Common, expected activity (filter out)
  • Tail: Rare, potentially suspicious (investigate)
Long Tail Process Analysis 
// Find processes in the long tail (bottom 1%)
let ProcessCounts = DeviceProcessEvents
| where TimeGenerated > ago(7d)
| summarize Count = count() by FileName
| extend Percentile = row_number() / todouble(count()) * 100;

ProcessCounts
| where Percentile > 99  // Bottom 1%
| join kind=inner (
    DeviceProcessEvents
    | where TimeGenerated > ago(1d)
) on FileName
| project TimeGenerated, DeviceName, FileName, ProcessCommandLine, Count
| order by Count asc

Interview Preparation

Stacking: Group data by a field and count occurrences, then sort to find outliers.

Use cases:

  • Find rare process names across environment
  • Identify unusual parent-child process relationships
  • Discover uncommon network destinations
  • Detect unique scheduled tasks or services

Key insight: Malicious activity is often rare - legitimate software appears on many systems, malware appears on few.
