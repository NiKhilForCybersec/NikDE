SAP Modules & Services

Key SAP modules and their security implications—FI, MM, HR, SD, and critical services that require monitoring.

Core SAP Modules

Module Function Security Concerns
FI (Financial Accounting) General ledger, AR/AP, assets Fraud, unauthorized payments, audit trail
CO (Controlling) Cost centers, profitability Budget manipulation, cost allocation fraud
MM (Materials Management) Purchasing, inventory Procurement fraud, vendor master changes
SD (Sales & Distribution) Orders, billing, pricing Pricing manipulation, unauthorized discounts
HR/HCM Personnel, payroll PII exposure, payroll fraud, ghost employees
PP (Production Planning) Manufacturing, BOM IP theft, production sabotage
Basis System administration Privileged access, configuration changes

RFC and Integration

⚠️ RFC Security Risks

Remote Function Calls (RFC) enable system-to-system communication but are frequently abused:

  • RFC destinations (SM59): Store credentials for remote systems
  • Trusted RFC: Can allow privilege escalation between systems
  • Function modules: Some allow direct table access (RFC_READ_TABLE)

Interview Preparation

User administration: SU01, SU10, PFCG (privilege changes)

System changes: SE38, STMS (code and transport)

Sensitive data: SE16, SE16N (direct table access)

Financial: F110 (payments), FK01/XK01 (vendor creation)
