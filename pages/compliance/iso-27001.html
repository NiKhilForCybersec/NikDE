ISO 27001:2022

International standard for Information Security Management Systems (ISMS), with focus on Annex A controls relevant to detection and monitoring.

ISO 27001 Overview

ISO/IEC 27001:2022 is the international standard for establishing, implementing, maintaining, and continually improving an Information Security Management System (ISMS). Certification demonstrates security commitment to customers and partners.

ISO 27001 (ISMS Requirements)

  • Management system requirements
  • Risk assessment process
  • Certification standard
  • Annex A: Control objectives

ISO 27002 (Control Guidance)

  • Implementation guidance
  • 93 controls in 4 themes
  • Best practice recommendations
  • Not certifiable directly

Auditors will request:

  • A.8.15: Log sources list, retention policy, sample logs, SIEM configuration
  • A.8.16: Detection rules documentation, alert procedures, monitoring dashboards
  • A.5.7: Threat intel sources, IOC feeds, threat reports
  • A.5.24-27: Incident response procedures, closed incident examples

Sample Audit Questions

Logging (A.8.15)

  • What systems generate security logs?
  • How long are logs retained?
  • How is log integrity protected?
  • Who has access to modify logs?

Monitoring (A.8.16)

  • How is anomalous behavior detected?
  • What alerts are generated?
  • How are alerts prioritized?
  • Show a recent alert investigation

Interview Preparation

Primary controls:

  • A.8.15 Logging: Foundational - what we collect
  • A.8.16 Monitoring: Core - how we detect
  • A.5.7 Threat Intelligence: Context for detection

Supporting controls:

  • A.5.24-28: Incident management lifecycle
  • A.8.7: Malware protection
  • A.8.20: Network security monitoring

Key changes:

  • Structure: 4 themes instead of 14 domains
  • Controls: 93 controls (down from 114)
  • New controls: Threat intelligence (A.5.7), monitoring (A.8.16), secure coding, cloud security
  • Attributes: Control types, properties, concepts for filtering

For detection: A.8.16 (Monitoring) is now explicit whereas 2013 had monitoring scattered across multiple controls.
